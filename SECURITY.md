# Security Policy

## Supported Versions

Security updates are provided for the latest code on the `main` branch.
Please upgrade to the latest release or commit before reporting issues that may
already be fixed.

## Reporting a Vulnerability

Please do **not** report security vulnerabilities in public GitHub issues.

Instead, report them privately by emailing:

- `opensource@kushalsrinivas.com`

Please include:

- A clear description of the vulnerability
- Steps to reproduce
- Affected versions, commits, or files
- Potential impact
- Any suggested remediation (if available)

You should receive an acknowledgment within 72 hours.

## Disclosure Process

After receiving a report, maintainers will:

1. Confirm receipt and begin triage
2. Validate and assess impact
3. Prepare and test a fix
4. Coordinate disclosure timing with the reporter
5. Publish a security fix and advisory notes

We aim to resolve critical issues as quickly as possible and keep reporters
informed throughout the process.

## Scope

This policy applies to:

- Source code in this repository
- Build and release configuration in this repository

Out of scope:

- Vulnerabilities in third-party dependencies that are unrelated to this
  project's usage or configuration
- Issues requiring physical device compromise outside normal app behavior
